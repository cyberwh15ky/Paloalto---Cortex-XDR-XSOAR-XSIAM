# 反向連線的「標準拓撲」
[ 你在家 / 外部 Client ]
          |
       Internet
          |
   Azure Public IP  (例如 52.x.x.x)
          |
   Azure VM (Linux, Nginx / Reverse Proxy)
          |
     IPSec / TLS Tunnel   ← Palo Alto 主動建立
          |
     Palo Alto (10.1.129.4)
          |
   GlobalProtect Portal / Gateway


# 真實流量路徑
[ 外部 Client ]
        |
        |  (TCP 443)
        v
[ Azure 公網 IP ]
        |
        |  (反向代理 / 中繼)
        v
[ IPSec / Tunnel ]
        |
        |  (Palo Alto 主動建立的連線)
        v
[ Palo Alto NGPW ]
        |
        v
[ GlobalProtect Portal / Gateway ]



### 關鍵點
> 外部永遠只看到 Azure 公網 IP  
> Palo Alto 永遠是 向外主動連線  
> 不存在「外部打你家裡」這件事
